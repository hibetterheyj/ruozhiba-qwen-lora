## 📋 初步方案框架

假设您选择 **"分类+解读复合任务"**（方向D），以下是完整方案：

### 一、SFT目标描述

```
目标：让模型成为"弱智吧文化专家"，能够：
1. 对输入帖子进行7分类打标
2. 输出梗的解读（隐喻/双关/逻辑分析）
3. 输出格式固定，无额外解释

示例格式：
{
  "category": "文字游戏",
  "interpretation": "利用'咏春'与'燕子叫'的谐音双关...",
  "confidence": 0.85
}
```

### 二、数据集处理

```
原始数据：tieba/best_100_all.json (7类已标注) + CQIA/ (解读成对)
修改策略：
1. 统一为instruction-output格式
2. 添加指令前缀："请分析以下弱智吧帖子的类型和含义："
3. 标准化输出为JSON格式
4. 数据增强：每条原文生成2-3种解读变体

预计规模：训练集3000-5000条，验证集500-800条
```

### 三、训练配置

```yaml
基础模型: Qwen2.5-1.5B-Instruct + Qwen2.5-7B-Instruct
微调方法: LoRA (r=16, alpha=32, dropout=0.1)
学习率: 2e-4
batch_size: 8 (1.5B), 4 (7B)
epochs: 3-5### 
max_seq_len: 512
硬件: 单卡80GB A100/H100
框架: HuggingFace Trainer + PEFT
```

### 四、评估设计

```
测试集：20条未参与训练的帖子（7类均匀分布）
对比内容:
- 微调前：原始模型输出（预期：格式混乱、分类不准）
- 微调后：SFT模型输出（预期：格式规范、分类准确）

评估维度:
1. 格式合规率（JSON是否有效）
2. 分类准确率（与人工标注对比）
3. 解读质量（人工打分1-5分）
```
